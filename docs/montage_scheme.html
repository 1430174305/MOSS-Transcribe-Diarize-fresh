MOSS 长音频说话人匹配 — 多随机 montage 投票方案

不用 speaker encoder、不用聚类阈值。对每相邻窗,生成多个随机 montage(随机片段 + 随机顺序 + 平滑拼接),每个过一遍 MOSS,统计谁和谁被 MOSS 判同标签的次数最多 → 多数投票定跨窗对应。随机化把单次拼接的顺序/噪声平均掉,匹配频次本身就是置信度。

三步:
1分段转写:切窗 → MOSS → 每窗 local [Sxx] 转写。
2片段池 + 随机 montage:按转写切每说话人语音段(过质量门),随机抽片段、随机顺序、平滑拼接成 K 个小 montage。
3批量重跑 MOSS + 投票:每 montage 得同标签关系,统计每对 (窗N-1 spk, 窗N spk) 的同标签频次 → 频次最高且显著 → 最终对应。

1. 参数

参数默认含义
K_montages12每对窗生成的随机 montage 数(自适应,见 §6)
frags_per_speaker_per_montage1-2每 montage 每说话人抽几段(少=montage 小=跑得快)
frag_min_seconds3.0片段最短(MOSS 认音色下限)
frag_max_seconds8.0片段上限
min_snr_db15.0片段 SNR 下限(质量门,可调)
max_overlap_ratio0.0片段与他人时段重叠占比上限(0=纯单说话人)
crossfade_ms60拼接 crossfade,消 click
inter_block_silence_ms300块间静音,给 MOSS 清晰说话人边界
tau_match0.6同标签频次 ≥ 此 + 显著领先 → 采信为同人
margin0.2最高 vs 次高频次的领先幅度下限

1 Step1 分段转写

长音频切 ≤90min 窗,每窗跑 MOSS → {offset, transcript(local [Sxx])}。本步只产出信息,不做匹配。

2 Step2 片段池 + 随机 montage

2.1 建片段池(带质量门,推荐)

对每 (窗 i, local 说话人 L),按转写 [start][Sxx]text[end] 切候选片段,过质量门入库:

维度门槛为什么
单说话人重叠占比 = 0(按转写时间戳,该片段时段无其他说话人)重叠=混合音,污染 MOSS 判同
时长3-8s太短认不准,太长占满 montage
SNR≥15 dB低噪音色被盖
无削波|peak|<0.99失真污染
质量门是推荐不是必须:不上质量门也能跑(纯随机抽原始片段),但每跑 MOSS 噪声大、投票收敛慢。上了质量门,每跑更干净,K 可以更小。

2.2 生成 K 个随机 montage

对 k in 1..K:
  对每 (窗, local) 说话人:从其片段池随机抽 frags_per_speaker_per_montage 段
  把所有抽中的片段随机打乱顺序(同人片段不连排)
  平滑拼接:块间 300ms 静音 + 60ms crossfade + 轻响度归一
  → montage_k (记录每块来自哪个 (窗, local))
关键:每 montage 体积小(每说话人 1-2 段 × 段长 3-8s → 几十秒级)→ 单次 MOSS 推理快;K 个并行/串行都跑得起。随机打乱顺序消除"位置偏置"(MOSS 可能把靠前的习惯性标 S01)。

3 Step3 批量重跑 MOSS + 投票(核心)

3.1 单个 montage 的同标签关系

montage_k 过 MOSS → 转写带 [Sxx]。按构造记录把每块映射回 (窗, local):该块时段内 MOSS 标签的众数 = 该 (窗, local) 在本 montage 的标签。于是本 montage 给出一组"同标签 = 同人"的等价类。

3.2 同标签频次统计

对每个候选对 (a_i, b_j)(a_i = 窗N-1 说话人,b_j = 窗N 说话人):

出现在一起的次数    n_ij = #{k | a_i 与 b_j 都出现在 montage_k}
被判同标签的次数    m_ij = #{k | a_i 与 b_j 都出现 且 二者同标签}
同标签频次          f_ij = m_ij / n_ij      # 条件频次(给定都出现)
必须用条件频次 f_ij = m_ij/n_ij,不能直接用 m_ij——否则"常出现的说话人"虚假占优。n_ij=0(从没一起出现)的对 → 无信号,跳过。

3.3 投票决策

对每个 a_i,看所有 b_j 的 f_ij:

  • 取 b* = argmax_j f_ij;
  • 采信条件:f_{i,b*} ≥ tau_match 且 f_{i,b*} − 次高 ≥ margin → 判 a_i ↔ b* 同人;
  • 不满足 → a_i 歧义,标记待人工/回退(见 §6)。

跨所有 a_i 汇总 → (窗N-1 local) ↔ (窗N local) 对应 → 链式传递得全局 ID。

4. 自适应 K 与可靠性诊断(投票的额外红利)

自适应 K:先跑 K0=8;若所有 a_i 的 top 频次都满足采信条件 → 提前停;否则补跑(翻倍 K)到 K_max(如 32)或信号收敛。
可靠性诊断:若 M 个说话人/窗,纯随机同标签的概率 ≈ 1/M。若所有对的 max f_ij < 诊断线(如 M=4 时 <0.4)→ 说明 MOSS 在 montage 上根本不一致(拼接输入退化)→ 整条 montage 路线不可用,回退 embedding。投票天然给出"这路能不能走"的信号。

5. 整体流程

长音频
  │
 ├─ Step1 分段转写:切窗 → MOSS → 每窗 {offset, transcript(local [Sxx])}
 │
 ├─ Step2 片段池 + 随机 montage:
 │     ├─ 按 transcript 切每说话人片段,过质量门(单说话人=0重叠/SNR≥15/3-8s)
 │     └─ for k in 1..K: 随机抽片段 + 随机打乱顺序 + 平滑拼接 → montage_k
 │
 └─ Step3 批量重跑 MOSS + 投票:
       ├─ 每 montage_k 过 MOSS → 块标签众数 → 同标签等价类
       ├─ 统计每对 (a_i,b_j): f_ij = 同标签次数 / 一起出现次数
       ├─ 决策:argmax f_ij,且 ≥tau_match + 领先 ≥margin → 同人
       ├─ 自适应:不够清晰补跑 K;全低 → 诊断 montage 不可用,回退 embedding
       └─ 链式传递 → 全局 ID → 重写 merged.txt

6. 输出

  • merged.txt:全局时间戳 + 全局说话人
  • vote_matrix.json:每对 (a_i,b_j) 的 n_ij/m_ij/f_ij(调试,看投票分布)
  • montages/:各 montage wav + 构造清单(块→(窗,local))
  • speaker_mapping.json:(窗,local)→全局ID + 置信度(=f_ij × 领先幅度)

7. 为什么这版最稳

风险单次 montage本版(多随机+投票)
MOSS 单次拼接不稳定直接错多次随机平均,错判被多数淹没
顺序/位置偏置有(靠前的易标 S01)随机打乱顺序,偏置平均为 0
低质片段污染该次直接错该次的错判不占多数;+质量门进一步降
置信度无(只有 0/1)f_ij 天然是置信度,可设门
路线是否可用不知道全低频次 = 自带诊断信号

8. 边界:某说话人没有干净独白片段怎么办

会议里常见:某说话人全程和别人重叠、或话太短,过不了 §1 的硬门控(单说话人=0 重叠)。若直接放弃,该说话人就没法跨窗匹配。解法:把硬门控改成分层兜底,总给他塞一段"能用的"进 montage,但降权 + 加 K。

8.1 分层片段选择(替代硬门控)

层overlap 占比SNR时长何时用投票权重
T1 干净=0≥153-8s首选1.0
T2 可用≤0.2≥102-8s该说话人 T1 空0.5
T3 兜底≤0.5≥81.5-8sT1/T2 都空0.25,标低置信
无———连 T3 都凑不出(段全<1.5s)不能 montage → 唯一 ID + 人工标记

对每说话人,按 T1→T2→T3 取他能达到的最高层最佳片段(层内按 SNR/overlap 排序)入 montage。硬门控(§1)实际是"只用 T1";这里是"T1 不够就降层兜底"。

8.2 低层说话人的三重补偿

  • 降权:T2/T3 片段入 montage,但其同标签投票按权重折算(T2×0.5、T3×0.25)计入 f_ij,不让噪声片段和干净片段同权;
  • 提高采信门槛:对该说话人的决策门槛更严(tau_match 提到 0.7、margin 提到 0.25)——因为信号本就噪,要求更清晰的领先才采信;
  • 加 K:涉及低层说话人的对,强制安排它俩在更多 montage 同现(定向采样),把 n_ij 撑大,让真对从噪声里冒头——这正是投票方案"多跑治噪"的本意。
为什么这样能work:低层片段虽含 overlap/噪声,但 MOSS 当初正是把这些段判给该说话人的,它对该主音色仍有识别力;多次随机 montage 里,真同人对的同标签频次仍会显著高于随机基线(1/M),只是需要更大 n_ij 才收敛。降权 + 加 K 就是把这个"需要更多样本"显式补上。

8.3 实在无解的兜底

  • 该说话人 T3 都凑不出 → 无法 montage → 给唯一全局 ID,标记待人工复核(不强行猜,避免误并);
  • 跨窗传播兜底:若该说话人在别的窗有干净片段(能被正常投票匹配),则他在本窗的全局 ID 可由邻窗匹配结果传递过来(同 local 标签链式传递),本窗无需独立 montage 他。

9. 限制

  • 算力:K 次 MOSS 推理/对窗。但每 montage 小(几十秒),单次快;K=12 可接受。K_max 上限防爆炸;
  • 条件频次前提:a_i 与 b_j 要在足够多 montage 里同时出现(n_ij 够大)才稳;片段池小的说话人 n_ij 低 → 该对信号弱,可强制安排它俩同现;
  • MOSS 标签抖动:即便同人,MOSS 也可能在一块里给俩标签 → 用众数挡;大面积抖动 → §4 诊断触发回退;
  • 跨多窗:逐对链式(1↔2→2↔3…);链式误差累积,可在关键对提高 K。
