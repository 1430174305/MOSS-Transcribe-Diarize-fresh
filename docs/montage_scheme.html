MOSS 长音频说话人匹配 — 高质量样本平滑拼接 montage 方案

不用 speaker encoder + 聚类,改用 MOSS 自身的 diarization 做跨窗匹配:对每相邻窗,从每个 (窗, local) 说话人选少量高质量语音块,平滑拼接成 montage 塞进一个 ≤90min 上下文,过一遍 MOSS → 同上下文内同人自动同标签 → 按"块来自哪个 (窗, local)"恢复全局映射。本方案的核心是"高质量块"的严格定义,直接规避污染块与非自然输入两个风险。

三步:
1分段转写:切窗 → MOSS → 每窗 local [Sxx] 转写。
2选高质量块 + 平滑拼接:按转写定位每说话人语音,过质量门控选 2-4 块,crossfade + 静音间隔拼成 montage。
3重跑 MOSS + 恢复映射:montage 过 MOSS → 每块标签 = 该 (窗, local) 的全局 ID。

1. 参数

参数默认含义
block_min_seconds3.0高质量块最短(够 MOSS 识别音色)
block_max_seconds10.0单块上限(避免一人占满 montage)
blocks_per_speaker3每 (窗,local) 说话人取多少块(代表性 vs montage 体积)
min_snr_db20.0块 SNR 下限(可降到 15)
min_voiced_ratio0.8块内实际语音占比下限
max_overlap_ratio0.0块与他人时段重叠占比上限(0=纯单说话人)
crossfade_ms60块边界 crossfade,消 click
inter_block_silence_ms300块间静音间隔,给 MOSS 清晰说话人边界
loudness_target_dbfs-23.0块响度归一目标(轻度,不重压缩)
montage_max_minutes85montage 总时长上限(留余量 <90min 上下文)

2. 高质量块的定义(核心)

对每个候选块 b(某说话人 L 在窗 i 内的一段连续语音 [t1, t2]),用硬门控(全过才收,任一不过则弃):

维度怎么测门槛为什么
单说话人(最关键)按转写时间戳查 [t1,t2] 内是否有其他说话人段重叠重叠占比 ≤ max_overlap_ratio(默认 0)重叠=混合音,污染该说话人音色,是 montage 头号失败源
时长块长 t2−t1block_min..block_max(3-10s)太短 MOSS 认不准音色;太长一人占满 montage
SNR语音帧 RMS / 噪声底 RMS(噪声底取全段静音分位)≥ min_snr_db(20,可放宽 15)低噪块音色被噪声盖,MOSS 易误判
语音占比VAD voiced 时长 / 块长≥ min_voiced_ratio(0.8)块里大半是静音/停顿,有效信息不够
无削波峰值 |sample|< 0.99削波失真污染音色
带宽采样率16k(切窗时已保证)窄带(电话 8k)音色信息少
"单说话人"是重中之重:用转写时间戳做硬过滤——块 [t1,t2] 内只要任何其他说话人段与之时间相交,就判为不纯,直接弃。这把"重叠语音切成混合块"这个最大污染源从源头掐掉。

选块策略:每 (窗, local) 说话人按 SNR 降序取 blocks_per_speaker(默认 3)个高质量块;若该说话人高质量块 < 1 → 标记无法 montage 匹配(回退到 embedding 或留独立 ID)。

3. 平滑拼接

把选出的块按 (窗, local, 块序号) 顺序拼成 montage,避免硬接产生 click / 让 MOSS 看清说话人边界:

  • 块间静音:插 inter_block_silence_ms(300ms)静音 → MOSS 能清晰切分"这是一段、下一段是另一人",标签更稳;
  • crossfade:块首尾各 crossfade_ms(60ms)做 cosine fade-in/out 与相邻静音/块交叠 → 消除拼接 click(硬切的高频瞬态会让 MOSS 误判);
  • 响度归一:每块 RMS 对齐到 loudness_target_dbfs(-23 dBFS)轻度归一,避免某说话人过小被淹没;不重压缩/不动态范围(MOSS 训练在自然电平上);
  • 块顺序:同 (窗, local) 的多块分散排(不要把一个说话人的 3 块连排,避免 MOSS 把"连排"当成一个超长 turn);不同窗交错排,让同人块在不同位置出现,利于 MOSS 一致识别。
体积控制:块数 = Σ(每说话人 blocks_per_speaker)。若两窗说话人多到 montage 超 montage_max_minutes,按"每说话人减到 2 块 → 1 块"递减排采样,仍超则该对窗回退 embedding。

4. 重跑 MOSS + 恢复映射

  1. montage 过一遍 MOSS(do_sample=False, max_new_tokens 给足),得 montage 转写 [t][Sxx]text[/t];
  2. 按块查标签:montage 构造时记录每块 (window, local, t_start_in_montage, t_end_in_montage);取该块时段内 MOSS 标签的众数 → 即该 (窗, local) 的全局 ID;(若一块被 MOSS 切成多段,取众数;若一块里出现两个标签,该块语音不纯,丢弃此块、该 (窗,local) 用其他块)
  3. 跨窗同一全局 ID 的 (窗, local) 即为同人 → 得 (窗, local) → 全局 ID 映射;
  4. 重写 merged.txt:全局时间戳 + 全局说话人。

5. 整体流程

长音频
  │
 ├─ Step1 分段转写:切窗 → MOSS → 每窗 {offset, transcript(local [Sxx])}
  │
 ├─ Step2 高质量选块 + 平滑拼接(对每相邻窗对 N-1/N):
  │     ├─ 按转写定位每 (窗,local) 语音段
  │     ├─ 硬门控筛高质量块(单说话人=0 重叠 / 3-10s / SNR≥20 / voiced≥0.8)
  │     ├─ 每说话人取 blocks_per_speaker 块(按 SNR 降序)
  │     └─ crossfade + 300ms 静音 + 轻响度归一 → montage(≤85min)
  │
 └─ Step3 重跑 MOSS + 恢复:
       ├─ montage 过 MOSS → 每块时段标签众数 = 该(窗,local)全局ID
       ├─ 同全局ID 的 (窗,local) = 同人
       └─ 重写 merged.txt(全局时间戳 + 全局说话人)

6. 输出

  • merged.txt:全局时间戳 + 全局说话人标签
  • montages/:每对窗的 montage wav + 构造清单(块→(窗,local)映射,调试)
  • montage_transcripts/:每 montage 的 MOSS 转写
  • speaker_mapping.json:(窗,local)→全局ID + 该 (窗,local) 用了哪些块

7. 与 embedding/聚类方案对比

维度本方案(高质量 montage 重跑 MOSS)embedding + 聚类
匹配主体MOSS 自身 diarization(SOTA)外部 speaker encoder
外部依赖无ECAPA / pyannote
阈值调参仅质量门(SNR/时长等,物理量好定)聚类阈值(需校准,encoder 相关)
重叠语音硬门控剔除(不纯块不入 montage)混合 embedding 污染(事后难剔)
非自然输入风险有(拼接独白)→ 用静音间隔+分散排序缓解无(只比音色,不在乎结构)
算力每对窗额外 1 次 MOSS(~2x)1x 推理 + 廉价 encoder
质量上限高(= MOSS diarization)取决于 encoder + 阈值

8. 限制与边界

  • 必须实测 MOSS 在拼接 montage 上是否还稳:即便加了静音间隔 + 分散排序,拼接独白仍非自然对话,MOSS 可能在边界处标签抖动 → 块标签众数法能挡一部分;若大面积抖动则该方案不可用,回退 embedding;
  • 相邻窗合并 ≤90min 时不必 montage:直接把两窗音频拼起来跑一次 MOSS 更省(一个 call 内标签一致);montage 价值在于两窗各自接近 90min、合并超限,但"只取高质量语音块"去静音后能塞进 90min;
  • 纯重叠/极噪说话人无高质量块→ 无法 montage 匹配,回退 embedding 或留独立 ID;
  • 跨多窗:逐对窗做 montage 链式传递(1↔2→2↔3→…),或对"全局种子说话人"建一次 montage(用首窗说话人作锚,后续每窗与之 montage)。
