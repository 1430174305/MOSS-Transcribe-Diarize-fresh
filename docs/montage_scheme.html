MOSS 长音频说话人匹配 — 多随机 montage 投票方案

不用 speaker encoder、不用聚类阈值。对每相邻窗,生成多个随机 montage(随机片段 + 随机顺序 + 平滑拼接),每个过一遍 MOSS,统计谁和谁被 MOSS 判同标签的次数最多 → 多数投票定跨窗对应。随机化把单次拼接的顺序/噪声平均掉,匹配频次本身就是置信度。

三步:
1分段转写:切窗 → MOSS → 每窗 local [Sxx] 转写。
2片段池 + 随机 montage:按转写切每说话人语音段(过质量门),随机抽片段、随机顺序、平滑拼接成 K 个小 montage。
3批量重跑 MOSS + 投票:每 montage 得同标签关系,统计每对 (窗N-1 spk, 窗N spk) 的同标签频次 → 频次最高且显著 → 最终对应。

1. 参数

参数默认含义
K_montages12每对窗生成的随机 montage 数(自适应,见 §6)
frags_per_speaker_per_montage1-2每 montage 每说话人抽几段(少=montage 小=跑得快)
frag_min_seconds3.0片段最短(MOSS 认音色下限)
frag_max_seconds8.0片段上限
min_snr_db15.0片段 SNR 下限(质量门,可调)
max_overlap_ratio0.0片段与他人时段重叠占比上限(0=纯单说话人)
crossfade_ms60拼接 crossfade,消 click
inter_block_silence_ms300块间静音,给 MOSS 清晰说话人边界
tau_match0.6同标签频次 ≥ 此 + 显著领先 → 采信为同人
margin0.2最高 vs 次高频次的领先幅度下限

1 Step1 分段转写

长音频切 ≤90min 窗,每窗跑 MOSS → {offset, transcript(local [Sxx])}。本步只产出信息,不做匹配。

窗口不需要重叠。montage 投票是按音色跨窗匹配(把两窗说话人片段塞进同一 montage 让 MOSS 判同),不靠"两窗同时间同说话人"的同时间证据。所以窗可以背靠背、VAD 切静音、无重叠——切窗更简单、不重复转写、文本合并退化为直接拼接(每窗拥有 [offset, end],相邻首尾相接)。重叠只是旧版(同时间 1:1)才需要,本版去掉。
唯一边际好处:重叠区让跨边界的说话人在两窗都多一点语音 → f_ij 稍稳,但非必需;想要这点边际可留小重叠(如 30s),否则直接无重叠最省。

2 Step2 片段池 + 随机 montage

2.1 建片段池(带质量门,推荐)

对每 (窗 i, local 说话人 L),按转写 [start][Sxx]text[end] 切候选片段,过质量门入库:

维度门槛为什么
单说话人重叠占比 = 0(按转写时间戳,该片段时段无其他说话人)重叠=混合音,污染 MOSS 判同
时长3-8s太短认不准,太长占满 montage
SNR≥15 dB低噪音色被盖
无削波|peak|<0.99失真污染
质量门是推荐不是必须:不上质量门也能跑(纯随机抽原始片段),但每跑 MOSS 噪声大、投票收敛慢。上了质量门,每跑更干净,K 可以更小。

2.2 生成 K 个随机 montage

对 k in 1..K:
  对每 (窗, local) 说话人:从其片段池随机抽 frags_per_speaker_per_montage 段
  把所有抽中的片段随机打乱顺序(同人片段不连排)
  平滑拼接:块间 300ms 静音 + 60ms crossfade + 轻响度归一
  → montage_k (记录每块来自哪个 (窗, local))
关键:每 montage 体积小(每说话人 1-2 段 × 段长 3-8s → 几十秒级)→ 单次 MOSS 推理快;K 个并行/串行都跑得起。随机打乱顺序消除"位置偏置"(MOSS 可能把靠前的习惯性标 S01)。

3 Step3 批量重跑 MOSS + 投票(核心)

3.1 单个 montage 的同标签关系

montage_k 过 MOSS → 转写带 [Sxx]。按构造记录把每块映射回 (窗, local):该块时段内 MOSS 标签的众数 = 该 (窗, local) 在本 montage 的标签。于是本 montage 给出一组"同标签 = 同人"的等价类。

3.2 同标签频次统计

对每个候选对 (a_i, b_j)(a_i = 窗N-1 说话人,b_j = 窗N 说话人):

出现在一起的次数    n_ij = #{k | a_i 与 b_j 都出现在 montage_k}
被判同标签的次数    m_ij = #{k | a_i 与 b_j 都出现 且 二者同标签}
同标签频次          f_ij = m_ij / n_ij      # 条件频次(给定都出现)
必须用条件频次 f_ij = m_ij/n_ij,不能直接用 m_ij——否则"常出现的说话人"虚假占优。n_ij=0(从没一起出现)的对 → 无信号,跳过。

3.3 投票决策

对每个 a_i,看所有 b_j 的 f_ij:

  • 取 b* = argmax_j f_ij;
  • 采信条件:f_{i,b*} ≥ tau_match 且 f_{i,b*} − 次高 ≥ margin → 判 a_i ↔ b* 同人;
  • 不满足 → a_i 歧义,标记待人工/回退(见 §6)。

跨所有 a_i 汇总 → (窗N-1 local) ↔ (窗N local) 对应 → 链式传递得全局 ID。

4. 自适应 K 与可靠性诊断(投票的额外红利)

自适应 K:先跑 K0=8;若所有 a_i 的 top 频次都满足采信条件 → 提前停;否则补跑(翻倍 K)到 K_max(如 32)或信号收敛。
可靠性诊断:若 M 个说话人/窗,纯随机同标签的概率 ≈ 1/M。若所有对的 max f_ij < 诊断线(如 M=4 时 <0.4)→ 说明 MOSS 在 montage 上根本不一致(拼接输入退化)→ 整条 montage 路线不可用,回退 embedding。投票天然给出"这路能不能走"的信号。

5. 整体流程

长音频
  │
 ├─ Step1 分段转写:切窗 → MOSS → 每窗 {offset, transcript(local [Sxx])}
 │
 ├─ Step2 片段池 + 随机 montage:
 │     ├─ 按 transcript 切每说话人片段,过质量门(单说话人=0重叠/SNR≥15/3-8s)
 │     └─ for k in 1..K: 随机抽片段 + 随机打乱顺序 + 平滑拼接 → montage_k
 │
 └─ Step3 批量重跑 MOSS + 投票:
       ├─ 每 montage_k 过 MOSS → 块标签众数 → 同标签等价类
       ├─ 统计每对 (a_i,b_j): f_ij = 同标签次数 / 一起出现次数
       ├─ 决策:argmax f_ij,且 ≥tau_match + 领先 ≥margin → 同人
       ├─ 自适应:不够清晰补跑 K;全低 → 诊断 montage 不可用,回退 embedding
       └─ 链式传递 → 全局 ID → 重写 merged.txt

6. 输出

  • merged.txt:全局时间戳 + 全局说话人
  • vote_matrix.json:每对 (a_i,b_j) 的 n_ij/m_ij/f_ij(调试,看投票分布)
  • montages/:各 montage wav + 构造清单(块→(窗,local))
  • speaker_mapping.json:(窗,local)→全局ID + 置信度(=f_ij × 领先幅度)

7. 为什么这版最稳

风险单次 montage本版(多随机+投票)
MOSS 单次拼接不稳定直接错多次随机平均,错判被多数淹没
顺序/位置偏置有(靠前的易标 S01)随机打乱顺序,偏置平均为 0
低质片段污染该次直接错该次的错判不占多数;+质量门进一步降
置信度无(只有 0/1)f_ij 天然是置信度,可设门
路线是否可用不知道全低频次 = 自带诊断信号

8. 边界:某说话人没有干净独白片段怎么办

会议里常见:某说话人全程和别人重叠、或话太短,过不了 §1 的硬门控(单说话人=0 重叠)。若直接放弃,该说话人就没法跨窗匹配。解法:把硬门控改成分层兜底,总给他塞一段"能用的"进 montage,但降权 + 加 K。

8.1 分层片段选择(替代硬门控)

层overlap 占比SNR时长何时用投票权重
T1 干净=0≥153-8s首选1.0
T2 可用≤0.2≥102-8s该说话人 T1 空0.5
T3 兜底≤0.5≥81.5-8sT1/T2 都空0.25,标低置信
无———连 T3 都凑不出(段全<1.5s)不能 montage → 唯一 ID + 人工标记

对每说话人,按 T1→T2→T3 取他能达到的最高层最佳片段(层内按 SNR/overlap 排序)入 montage。硬门控(§1)实际是"只用 T1";这里是"T1 不够就降层兜底"。

T1 的来源就是"独白子区间":对每说话人的段(含重叠区里和他交叠的段),扣掉与其他说话人交叠的部分,剩下 overlap=0 的子区间即 T1(见 §8.4)。所以即便某说话人出现在重叠区,只要他有过独白瞬间,就能抠出 T1 干净片段,不必降层。

8.2 低层说话人的三重补偿

  • 降权:T2/T3 片段入 montage,但其同标签投票按权重折算(T2×0.5、T3×0.25)计入 f_ij,不让噪声片段和干净片段同权;
  • 提高采信门槛:对该说话人的决策门槛更严(tau_match 提到 0.7、margin 提到 0.25)——因为信号本就噪,要求更清晰的领先才采信;
  • 加 K:涉及低层说话人的对,强制安排它俩在更多 montage 同现(定向采样),把 n_ij 撑大,让真对从噪声里冒头——这正是投票方案"多跑治噪"的本意。
为什么这样能work:低层片段虽含 overlap/噪声,但 MOSS 当初正是把这些段判给该说话人的,它对该主音色仍有识别力;多次随机 montage 里,真同人对的同标签频次仍会显著高于随机基线(1/M),只是需要更大 n_ij 才收敛。降权 + 加 K 就是把这个"需要更多样本"显式补上。

8.3 实在无解的兜底

  • 该说话人 T3 都凑不出 → 无法 montage → 给唯一全局 ID,标记待人工复核(不强行猜,避免误并);
  • 跨窗传播兜底:若该说话人在别的窗有干净片段(能被正常投票匹配),则他在本窗的全局 ID 可由邻窗匹配结果传递过来(同 local 标签链式传递),本窗无需独立 montage 他。

8.4 重叠区域怎么算匹配(关键修正)

前提修正:MOSS 对重叠语音能给出每个说话人各自的段(时间可交叠),不是"一个混合块判给主说话人"。所以重叠区不是识别问题,是怎么从这些交叠段里给每说话人抠出干净音频的问题。

① MOSS 对重叠区的输出长这样

[10.0][S01]大家好[15.0]       ← S01 段 [10,15]
[12.0][S02]抱歉打断[16.0]      ← S02 段 [12,16]  (和 S01 在 [12,15] 交叠)

两人各有段、时间交叠。解析器按序读,不强制单调,能正常解析出两个时间交叠的段。

② 抠独白子区间(核心操作)

对每说话人 L 的段 [s,e],扣掉所有其他说话人段覆盖的时间,得到 L 的"独白子区间"=[s,e] 里不被任何人覆盖的部分:

S01 [10,15] 与 S02 [12,16] 交叠 →
  S01 独白子区间 = [10,12]   ← S02 还没开口 → 干净 S01 音频
  S02 独白子区间 = [15,16]   ← S01 已结束   → 干净 S02 音频
  [12,15] = 真重叠(两人同时,混合音频,丢弃不用)
每说话人从自己的交叠段里抠独白子区间 → 这些子区间 overlap=0(干净)→ 直接当 T1 片段入 montage → 正常音色投票。重叠区不再是"特殊块"——它只是段在时间上交叠,扣掉交叠部分,每人都能拿到自己的干净音频,只要他有过哪怕一瞬独白(先开口/后收尾)。

③ 匹配逻辑(和其他说话人完全一样)

  1. 解析转写(允许段时间交叠);
  2. 每说话人算各段的独白子区间(扣掉与其他说话人交叠的部分);
  3. 独白子区间 → T1 干净片段 → montage 投票,无特殊处理;
  4. 真重叠的混合区间 [12,15] 直接丢弃,不入 montage(它谁的干净音频都不是)。

④ 极端:某说话人零独白子区间

该说话人所有段都被其他说话人完全覆盖 → 抠不出任何干净音频(从没独白过)→ 进 §8.2 的 T2/T3 兜底(用混合段、降权 + 加 K)或 §8.3 消去法/人工。这是少数极端,大多数重叠里每人都有"先开口/后收尾"的独白瞬间可抠。

⑤ 最小例子(2 窗 × 2 人,用独白子区间)

  • 窗1:A(有独白子区间 A)、B(段和 A 交叠,但 B 有独白子区间 B=收尾那段)
  • 窗2:C(=A)、D(=B),同理各有独白子区间

montage 里放的是 A、B、C、D 的独白子区间音频(都 overlap=0),MOSS 按音色判同标签 → A↔C、B↔D 各自投票,无交叉污染。重叠不再制造假票,因为混合区间根本没进 montage。

一句话:MOSS 能给重叠区每说话人各自的段;匹配时从每人段里扣掉与他人交叠的部分、留独白子区间作干净片段,混合的真重叠区间丢弃。重叠区说话人和普通说话人一样走音色投票,不需要"票归主说话人/消去法"那套(那套是旧错误假设下的补救)。

8.5 零独白子区间的说话人怎么跨窗匹配(真正的难点)

前提:说话人 L 在某窗里所有段都被其他说话人完全覆盖,抠不出任何独白子区间(从没独白过)→ 没有干净音频 → 不能直接音色投票。这种情况按下面优先级链式兜底匹配(§8.4 处理的是"有独白子区间的重叠区说话人",本节是"连独白子区间都没有的")。

① 消去法(首选,前提:两窗说话人集合对应)

  1. 先把有独白子区间的说话人全部音色投票匹配掉(可靠);
  2. 每窗剩下"零独白"的未匹配集合 R = 该窗零独白说话人;
  3. 若相邻两窗 |R_{N-1}| = |R_N|(人数相等)→ 按"剩余即同人"配对(L↔L'),标低置信(间接/消去),无直接语音证据;
  4. 人数不等 → 消不去,进 ②。
消去法的逻辑:同一拨人开会,干净匹配掉能配的,剩下的零独白者按集合对应配对。前提是两窗是同一拨人(常见于会议中途,无新人加入/离开)。

② 结构共现(host-based,无集合对应时)

  • L 总和某个有独白的"宿主" X 交叠(L 的每次 overlap 都落在 X 上);
  • 若 X 已干净匹配到 X',且 L' 在另一窗也总交叠 X' → L↔L'(L 是"覆盖 X 的那个人",L' 是"覆盖 X' 的那个人");
  • 多宿主(L 同时覆盖 X 和 Y)→ 结构不唯一,降置信;宿主未匹配 → 此路不通,进 ④。
结构共现用"谁和谁总一起讲"的 diarization 结构作信号,不需音频。脆弱点:L 在 w1 覆盖 X、在 w2 却覆盖 Y(X≠Y)→ 结构对不上(虽是同人)→ 此路失效,得靠 ① 或 ④。

③ 混合段降权投票(微弱直接信号,通常不单独采信)

  • 把 L 的混合段(含 overlap 的)作 T3 兜底片段入 montage,极重降权(×0.25)+ 定向加 K(强制 L 与候选同现);
  • 若 L 的声音在混合里仍可辨,多次随机投票后真对的 f_ij 可能微弱冒头 → 极低置信,一般仅作 ①/② 的旁证,不单独定案;
  • 多数情况信号太弱(L 被主说话人盖住)→ 不采信。

④ 跨窗传播(常被忽略但很有效)

  • 若 L 在别的窗(N-2 或 N+1)有独白子区间 → 那窗能干净匹配 L → 全局 ID 链式传到本窗(本窗 L 直接继承该全局 ID,无需独立匹配);
  • 即"某人在本窗全程被盖,但在邻窗说过独白"→ 用邻窗的匹配结果兜底。多数零独白情况都能靠这个解掉。

⑤ 实在无解

  • ① 消不去、② 结构不通、④ 邻窗也没独白 → 唯一全局 ID + 人工复核标记(不强行猜,避免误并)。
优先级:① 消去(最稳,前提集合对应)→ ④ 跨窗传播(邻窗有独白就用)→ ② 结构共现(同时间总一起讲)→ ③ 混合投票(微弱信号,仅旁证)→ ⑤ 人工。
关键认知:零独白者没法靠本窗音色定,但会议是同一拨人这个先验让"消去法 + 跨窗传播"在大多数情况能解;真正无解(全程被盖、邻窗也没独白、人数还对不上)是极少数,留人工。

⑥ 最小例子(2 窗 × 3 人,窗2 的 C 零独白)

窗1(都有独白):

[1.0][S01]大家好我们开始[5.0]          ← A,独白
[6.0][S02]我先过一下进度[10.0]         ← B,独白
[11.0][S03]那个bug我修了[15.0]        ← C,独白

窗2(A'、B' 独白;C' 全程被 B' 盖住,零独白):

[22.0][S01]继续讨论性能[26.0]              ← A',独白(=A)
[29.0][S02]测试报告在这我补充几点[35.0]     ← B',独白 [29,35](=B)
[30.0][S03]对性能我补充[34.0]               ← C' [30,34],完全落在 B' [29,35] 里 → 零独白子区间(=C)

若还有窗3,且 C 在窗3 有独白 → C 在窗3 能干净音色匹配 → ④ 跨窗传播把全局 ID 传回窗2 的 C',置信度比消去法高(从"低置信间接"升到"中高置信")。即"本窗被盖,但别处说过独白"就用别处的匹配结果。

全局注册表设计(解加入/离开/再出现——本节重点)

最需要解的是"再出现"。加入=开新 ID(简单)、离开=不延续(简单),但消失再出现要求"人不在中间窗、靠旧片段把他认回来"——这就逼着注册表保留他的旧片段,不能只留近期代表(只留近期=把他丢了)。

两种维护法:

  • A. 固定容量 + 替换刷新:每全局说话人保留 k 个片段,新窗匹配后用新片段替换旧的。省内存,但有把 canonical 旧 voiceprint 换丢的风险 → 再出现时认不出。
  • B. 全保留 + 加权 top-k 选取(推荐):所有片段都留(片段秒级,5h 会议几百段=几 MB,可忽略),每次匹配按"可靠性 + 时近度"加权排序取前 k 个拼 montage。B 严格优于 A:不丢信息,又通过排序兼顾 drift(近期)和再出现(旧片段也保留)。

B 的加权分(挑代表片段时,对该全局说话人所有片段排序):

score = reliability + λ · recency
reliability = norm(min(dur, D_cap)) + (1 − overlap_ratio) + snr_norm + stability
recency     = exp(−(t_query − t_frag) / half_life)   # 近期高,旧的非零(不被压没)
  • 可靠性是主 driver(长 / 干净 / 稳 / 高 SNR);
  • recency 软偏好近期(drift 鲁棒),λ 取小 → 不会把再出现的旧片段压没(旧的是该说话人仅有候选,照样被选)。

匹配流程(每来一个新窗):

  1. 新窗每说话人 L 取其高质量片段;
  2. 对注册表每个全局说话人 G,按 score 取 G 的 top-k 片段作"代表";
  3. L 的片段 + 各 G 的代表 → 拼 montage → MOSS 投票 → L 命中某 G 或都不中;
  4. 命中 → L 的片段并入该 G(注册表增长,全保留);都不中 → 开新 G,L 片段入库。

三种情况自动覆盖:

情况B 怎么解
加入(窗N 多人)L 与所有 G 都不中 → 开新 G 入库
离开(窗N 少人)L 不出现 → 无匹配;该 G 沉默保留(片段还在,等再出现)
再出现(1有/2无/3有)旧片段被 score 选出(即便 recency 低,仍是该 G 唯一候选)→ 认回,继承原 ID
drift(5h 音色变)近期片段 score 高 → 代表偏近期 → 匹配对当前音色更准;旧片段仍留作兜底
容量上限(防极端):个别话痨说话人可能积累上千片段。可加软上限(如每 G 保留 reliability 最高的 top-N + 近期 M 个),但 N 要足够大(几百)不丢再出现能力。5h 会议一般不触发。

窗内一致性预检(跨窗前先修 MOSS 单窗错误)

跨窗匹配前,先在每窗内部跑一次小 montage(同窗各 local 标签互拼):若两 local 标签在多次随机 montage 里总被判同标 → 实为同人被 MOSS 拆开 → 合并成一个 local 再跨窗。这把"MOSS 一人拆两标签"的错误在跨窗前修掉,避免后续匹配混乱。

8.7 注册表匹配的运行机制(补全前面没说清的)

① 首窗引导(bootstrap)

注册表初始为空 → 首窗每个说话人直接开新全局 ID,片段入库。从第二窗起才走"匹配 vs 注册表"流程。

② 一次匹配的 montage 构造:per L,不是 per (L,G)

对新窗说话人 L,一个 montage = L 的 2-3 片段 + 注册表每个 G 的 top-k 代表。所有 G 代表都进同一个 montage(不是 L 和每个 G 单独拼),让 MOSS 在同一上下文里给 L 和所有候选 G 一起判标:

montage_k = [L 的片段...] + [G1 代表...] + [G2 代表...] + ... + [Gm 代表...]
            (m 个 G × k 代表 × ~5s + L ≈ 3min,远 <90min,装得下)
跑 MOSS → L 的片段得某标签 X;哪个 G 的代表也得 X → L 匹配该 G
  • L 与所有 G 在每个 montage 都同现 → n_{L,G} = K(总同现),f_{L,G} = (L 与 G 同标签的次数)/K;
  • L 的匹配 = argmax_G f_{L,G}(真对 f 高,其他 ≈ 1/M 随机基线);
  • 算力:每 L 跑 K 个小 montage(每个 ~3min 音频,MOSS RTF~0.06 → 单次 <0.2s),每窗几秒~几十秒,可接受。

③ 一对多冲突(多个 L 都强匹配同一个 G)

  1. 先做窗内一致性预检(§8.6):L1、L2 是否同人被 MOSS 拆开(montage 里 L1、L2 总同标)→ 是则合并成一个 L,再匹配;
  2. 若确为两人都强匹配 G(罕见,G 被过匹配)→ 取 f 更高者匹配 G,另一个标"待定" → 加 K 或人工;
  3. 极端:G 本身是 MOSS"两人合一"的错误 → §8.6 拆标 + 人工。

④ 都不中(L 是新人)

所有 f_{L,G} ≈ 1/M(都贴随机基线,无显著高者)→ 开新全局 G,L 片段入库。

⑤ 与 step1 切窗的衔接(代码侧)

本方案不需要窗间重叠(§1 注)。scripts/windowed_inference.py 已加 --no_overlap 开关:开启后走背靠背切窗(每窗在 VAD 静音最近 target 处切,下窗从该静音开始,无重叠),文本合并退化为直接拼接。默认关(保留重叠给旧版同时间 1:1)。

8.8 一对多 / 多对一冲突怎么处理(投票后的消解)

多次投票后出现:一对多(一个新窗 L 对多个全局 G 都高 f,L 像 G1 也像 G2)、多对一(多个新窗 L1,L2 对同一 G 都高 f)。先判"多"的一方是同人被拆还是不同人音色近,再用全局最优分配兜底。

① 先判"多"的一方是不是同人(是→合并,变一对一)

  • 多对一(L1,L2→G):把 L1、L2 片段互拼 montage 投票 → f(L1,L2) 高 → 同人被 MOSS 拆开 → 合并 L1,L2 成一个 L再匹配(变一对一);f 低 → 确为不同人 → 进 ②。
  • 一对多(L→G1,G2):把 G1、G2 代表互拼 → f(G1,G2) 高 → 同人被注册表过合 → 合并 G1,G2再匹配;f 低 → 确为不同人 → 进 ②。
这一步把"同人被拆/过合"的 MOSS 错误先修掉,剩下的冲突才是"真·不同人音色近"。

② 全局最优分配(Hungarian 二分匹配,核心)

把"新窗 L 集 × 全局 G 集"的 f 矩阵当二分图,跑 Hungarian 求总 f 最大的一对一分配(每 L 至多一个 G、每 G 至多一个 L),带质量门 f ≥ tau_match:

对每 L:  匹配到 argmax_G f_{L,G}(若 ≥tau) → 该 G;否则 L 是新人
约束:  每个 G 只能被一个 L 认领(多对一自然消解:取 f 最高者,其余 L 落空)
        每个 L 只能认一个 G(一对多自然消解:取 f 最高者,其余 G 不被该 L 占)
  • 多对一消解:G 只被 f 最高的那个 L 认领;其余 L(确为不同人)→ 对它们而言 G 已被占 → 若对其他 G 也 开新全局 ID(它们是音色相近的不同人)。
  • 一对多消解:L 只认 f 最高的 G;其余 G 不被 L 占(它们由各自真正的对应 L 去认,或沉默)。

③ 近平手(f 差 < margin)

  • 同一 L 对 G1/G2 的 f 几乎并列 → 定向加 K(强制 L 与这两个 G 多次同现)破并列;
  • 仍并列 → 标人工复核,不强行二选一(避免误并)。

④ 残留真·音色近

  • Hungarian 把高 f 的一方判给 G,另一方无人认领 → 开新全局 ID(不同人,只是音色近);
  • 极端:两人音色极近,Hungarian 也分不开 → 接受可能误并,标低置信 + 人工复核。

9. 限制

  • 算力:K 次 MOSS 推理/对窗。但每 montage 小(几十秒),单次快;K=12 可接受。K_max 上限防爆炸;
  • 条件频次前提:a_i 与 b_j 要在足够多 montage 里同时出现(n_ij 够大)才稳;片段池小的说话人 n_ij 低 → 该对信号弱,可强制安排它俩同现;
  • MOSS 标签抖动:即便同人,MOSS 也可能在一块里给俩标签 → 用众数挡;大面积抖动 → §4 诊断触发回退;
  • 跨多窗:逐对链式(1↔2→2↔3…);链式误差累积,可在关键对提高 K。
