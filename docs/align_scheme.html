MOSS 长音频说话人对齐方案(三步法)

把长音频(超 90min 上下文)的说话人一致化拆成三步,step1/step2 只做信息汇总,所有算法集中在 step3 对齐。本版暂不使用前后两段重叠区的 1:1 对齐逻辑,改走"每窗稳定提取 → 不足重提取 → 全局相似度/聚类"。

总览:
1分段转写:长音频切段,每窗跑 MOSS,得到带 local [Sxx] 的转写。
2滑窗提取 embedding:按转写定位每个说话人语音区,滑窗过 speaker encoder,得大量 sub-embedding。
3对齐(核心):稳定提取+去不稳 → 不足/缺失重提取 → 全局相似度/聚类 → 全局说话人 ID。

1. 参数

参数默认含义
sub_window_seconds3.0step2 滑窗长度
sub_step_seconds1.0滑窗步长(重叠 2/3)
min_sub_speech_seconds1.0子窗内有效语音 < 此则跳过(不 embed)
stability_threshold0.7step3a:子 embedding 到中位数的余弦 ≥ 此才保留(稳核)
min_stable_count3稳核子 embedding 数 < 此 → 判为"不足",进 3b 重提取
min_stable_speech_seconds5.0稳核总语音 < 此 → 判为"不足"
reextract_min_solo_seconds3.03b:只用 ≥ 此的"完全包含"独白段重提取
cluster_threshold0.55step3c 聚类余弦阈值(需校准)

1 Step1 分段转写

长音频切成 ≤90min 的窗(VAD 切静音或固定长),每窗独立跑 MOSS 推理。

  • 每窗输出 [start][Sxx]text[end] 格式,时间戳为窗内 local,说话人标签 [Sxx] 是每窗独立从 S01 重新编号(窗1的 S01 和窗2的 S01 不一定是同一人——这正是 step3 要解决的);
  • 记录 {offset, transcript};offset = 窗在全音频里的起始秒,用于还原全局时间戳。

本步只产出信息,不做任何对齐。窗之间是否重叠不影响 step3(本版不用重叠 1:1)。

2 Step2 滑窗提取 embedding

对每窗,按转写段 [start][Sxx]text[end] 定位每个 local 说话人的语音区间,在区间上滑窗过 speaker encoder。

对窗 i 的每个 local 说话人 L:
  语音区 = {(seg_start, seg_end) | seg.speaker == L}  (该窗 local 时间)
  for t in seg_starts, step=sub_step:
      w = [t, t+sub_window]
      if 该窗内 w 内 L 的有效语音 >= min_sub_speech_seconds:
          emb = speaker_encoder(audio[ w + offset ])   # 切该窗音频对应区
          保存 (i, L, t, emb, dur)  # 一个 sub-embedding
为什么要滑窗而不是整段一个 embedding:一个说话人在一窗里有多个 sub-embedding → 才能在 step3a 算"稳定性"(段内音色是否一致)、剔除离群(混入他人/噪声的子窗)。整段一个 embedding 无法做稳定性判断。

encoder 选型:SpeechBrain ECAPA-TDNN(纯 pip,起步推荐)或 pyannote/embedding(WavLM,质量更高)。embedding 归一化到单位长度。不能用 Whisper encoder(编码内容不编码音色)。

3 Step3 对齐(核心,所有算法在此)

输入:每 (窗 i, local 说话人 L) 的一组 sub-embeddings。输出:(窗, local) → 全局说话人 ID。

3.1 稳定提取 + 去除不稳定

对每个 (i, L):

  1. 算中位数向量 m = normalize(median(sub_embs));
  2. 每个 sub-embedding 的稳定度 s_k = cos(emb_k, m);
  3. 保留 s_k ≥ stability_threshold 的子 embedding → 稳核 (stable core);
  4. 该说话人的稳定度 = 稳核内子 embedding 两两余弦的中位数(高=音色一致、可信;低=混/噪)。
被剔除的子 embedding 多半是混入他人 / 突发噪声 / 极短窗,本步把它们清掉,不让它们污染后续聚合。

3.2 不足 / 缺失的重提取(完全包含的说话人)

若 (i, L) 的稳核不满足 count ≥ min_stable_count 且 总稳核语音 ≥ min_stable_speech_seconds → 判为不足,重提取:

  1. 在该窗转写里找 "完全包含"的独白段:该段只有 L 一个说话人、无重叠、段长 ≥ reextract_min_solo_seconds;
  2. 对这些干净独白段重新切音频、重新滑窗 embed(可用更长 sub_window,提升信号);
  3. 再跑一次 3.1 稳定过滤,合并进稳核。
还不足怎么办:该 (i, L) 仍不足 → 标记为 低置信向量(不作为聚类中心,但可在 3.3 被分配到最近的高置信簇)。极端没数据 → 该 local 标签无法对齐,保留为独立 ID(待人工)。
要点:先把每窗每个说话人的 embedding 提稳、提全;不稳的删、不够的用干净独白段补。这一步是后面聚类能否正确的前提。

3.3 全局相似度 / 聚类

每个 (i, L) 聚合成一个稳定向量 v_{i,L}(稳核的加权均值,权重=稳定度),并带置信度 c_{i,L}(= 稳定度 × 稳核规模因子)。

  1. 算所有 (i, L) 两两余弦 cos(v_a, v_b);
  2. 聚类:以 cos ≥ cluster_threshold 为边建图,取连通分量(或用 agglomerative + 平均链接 + 阈值剪枝)。高置信向量优先作簇锚;
  3. 低置信向量 → 分配到最近的簇(余弦最高者),若都 < 阈值 → 开新簇(独立 ID);
  4. 冲突(多对一)→ 取最高余弦,其余按次高分配或开新簇。

每个簇 = 一个全局说话人 ID(S01, S02, …)。得到 (窗 i, local L) → 全局 ID 映射。

4. 整体流程

长音频
  │
 ├─ Step1 分段转写:切窗 → MOSS → 每窗 {offset, transcript(local [Sxx])}
 │
 ├─ Step2 滑窗提取:按 transcript 定位每 (i,L) 语音区 → 滑窗 speaker encoder
 │     ↓ 每 (i,L) 一组 sub-embeddings
 │
 └─ Step3 对齐(算法集中):
       ├─ 3.1 稳定提取:中位数 m → 保留 cos(emb,m)≥τ 的稳核,删离群
       ├─ 3.2 不足重提取:稳核不够 → 找"完全包含"独白段重 embed;还不足→低置信
       └─ 3.3 全局聚类:v_{i,L}=稳核加权均值 → 两两余弦 → 连通分量聚类 → 全局 ID
             ↓ (窗,local)→全局ID 映射
       → 重写 merged.txt:全局时间戳 + 全局说话人

5. 输出

  • merged.txt:全局时间戳 + 全局说话人标签(S01/S02… 跨窗一致)
  • embeddings.json:每 (窗,local) 的稳核向量 + 置信度(调试)
  • speaker_mapping.json:(窗,local)→全局 ID + 置信度 + 稳定度
  • clusters.json:聚类结果(每簇成员、余弦矩阵)

6. 与旧方案(重叠 1:1)的差异

维度旧方案(scheme.html)本方案
对齐依据重叠区同时间 1:1 对应全局 embedding 相似度/聚类
是否用重叠区是,必须有重叠且两窗都转写不用(暂不考虑重叠 1:1)
稳定性靠重叠同时间证据靠滑窗 sub-embedding 稳核过滤
不足处理短段降权"完全包含"独白段重提取
复杂度对齐逻辑分散(切窗+重叠+1:1+回退)算法集中 step3,step1/2 只汇总

7. 边界与限制

  • 聚类阈值 cluster_threshold 需按你的 encoder + 域校准(不同 encoder 余弦尺度不同);
  • 同一人音色跨窗漂移(疲劳/换麦)可能被聚成两簇 → 阈值偏松会合并不同人,偏紧会拆同一人,需权衡;
  • 转写里说话人标签错误(MOSS 把两人合一/一人拆二)会污染该窗 embedding,3.1 的稳核过滤能挡一部分;
  • 本版不用重叠 1:1,实现简单,但放弃了"同时间强证据";若后续聚类效果不够,可再叠加重叠 1:1 作约束。
