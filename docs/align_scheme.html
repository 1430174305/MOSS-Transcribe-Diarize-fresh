MOSS 长音频说话人对应关系求解方案

把长音频(超 90min 上下文)的跨窗说话人一致化拆成三步,step1/step2 只汇总信息,所有算法集中在 step3 的对应关系求解。本版不用前后两段重叠区的 1:1 对齐,改走"生成式匹配概率 → 最小违例对应优化 → 变分 E-M 精修 → 最优传输仲裁"的有数学骨架的求解。

总览:
1分段转写:切窗 → MOSS → 每窗带 local [Sxx] 转写。
2滑窗提取 embedding:按转写定位每说话人语音区,滑窗过 speaker encoder → 每 (窗,local) 一组 sub-embedding。
3对应关系求解(核心):稳健原型 → 不足重提取 → 生成式匹配概率 → 最小违例分割(correlation clustering)→ 变分 E-M → 最优传输仲裁 → 全局说话人 ID。

1 Step1 分段转写

长音频切成 ≤90min 的窗(VAD 切静音或固定长),每窗独立跑 MOSS。

  • 每窗输出 [start][Sxx]text[end],时间戳为窗内 local,说话人标签 [Sxx] 每窗独立从 S01 重新编号(跨窗不保证同人);
  • 记录 {offset, transcript};offset 还原全局时间戳。

本步只产出信息,不做任何对应关系求解。窗是否重叠不影响 step3。

2 Step2 滑窗提取 embedding

对每窗,按转写段定位每个 local 说话人语音区间,在区间上滑窗过 speaker encoder。

对窗 i 的每个 local 说话人 L:
  语音区 = {(seg_start, seg_end) | seg.speaker == L}
  for t in seg_starts, step=sub_step:
      w = [t, t+sub_window]
      if w 内 L 的有效语音 >= min_sub_speech_seconds:
          emb = speaker_encoder(audio[ w + offset ])   # 归一化
          存 (i, L, t, emb, dur)

encoder:SpeechBrain ECAPA-TDNN(起步)或 pyannote/embedding(WavLM),均归一化。不能用 Whisper encoder。滑窗目的是给 3.1 提供冗余样本以做稳健估计。

3 Step3 对应关系求解(核心,所有算法在此)

输入:每实体 e = (窗 i, local L) 的一组 sub-embeddings。输出:e → 全局说话人 ID。把"哪些实体同人"建模为带概率边的图分割优化问题。

3.1 稳健原型 (Robust Prototype) — M-估计

对每实体 e,把 sub-embeddings 蒸成稳健原型 p_e + 置信度 c_e:

  1. 迭代重加权 M-估计:以当前均值 m 为参照,赋 Tukey/biweight 权 w_k = ψ(cos(emb_k, m))(亲和高→权大,离群→权趋零),重算 m = Σ w_k emb_k / Σ w_k,迭代收敛 → 原型 p_e;
  2. 稳核:剔 cos(emb_k, p_e) < stability_threshold 的离群;
  3. 稳健度 ρ_e = 稳核内子向量到 p_e 的平均亲和;
  4. 置信度 c_e = g(ρ_e, 稳核规模, 稳核总语音);
  5. 同时估计同人变异协方差 Σ_within:所有实体稳核内的条件散度平均(全局共享)。

3.2 不足 / 缺失的重提取(完全包含的说话人)

若稳核不满足 count ≥ min_stable_count 且 总语音 ≥ min_stable_speech_seconds → 判不足:

  1. 在该窗找 "完全包含"独白段(只 L、无重叠、段长 ≥ reextract_min_solo_seconds);
  2. 对干净独白段重切音频、重滑窗 embed,再过 3.1 稳核过滤并入稳核。
仍不足 → c_e 极低,标记为待分配(不主导 3.4 优化,在 3.5/3.6 被求解)。

3.3 生成式匹配概率 (Generative Match Posterior)

把裸余弦升级为校准后的同人后验,建模为两个生成模型的似然比:

同说话人模型 H_same:  emb ~ N(p, Σ_within)       # 同人音色波动
异说话人模型 H_diff:  emb ~ N(p, Σ_between)      # 不同人差异(全局散度估计)

似然比 LLR(e,f) = log p(p_e, p_f | H_same) − log p(p_e, p_f | H_diff)
同人后验 P_same(e,f) = σ( LLR + log(π/(1−π)) )   # π = prior_same,σ=sigmoid

边权不再是"相似度",而是在生成式模型下的同人后验概率 q_ef = P_same(e,f)。Σ_within/Σ_between 从数据自适应估计(也可外接预训练值)。

3.4 最小违例对应分割 (Minimum-Disagreement Partition)

这是本方案的核心优化。建带概率边的图 G=(V,E),V=实体,边权 q_ef(同人后验)。求分割 Π(每部分=一个全局说话人)使期望违例代价最小:

min_Π  Cost(Π) =
   Σ_{(e,f) 被拆开}   q_ef        # 强同人却拆开 → 代价 q_ef
  + Σ_{(e,f) 被合并}   (1 − q_ef)  # 强异人却合并 → 代价 1−q_ef

即:最小化"判错"的期望次数。这是 correlation clustering(无预设说话人数 K)的标准目标。

求解:

  • solver=pivot:Pivot 局部搜索(Ailon–Charikar–Newman),O(n²),3-近似保证,大实例用;
  • solver=ilp:整数线性规划精确解(branch-and-bound),实体数 ≤ ~50 时可用,得全局最优。
这步把"谁和谁同人"从"启发式分层匹配"升级为一个有目标函数的优化问题——直接最小化判错期望,无人工阈值(τ_spawn/match/assign 退化为生成式后验的自然产物)。

3.5 变分 E-M 精修 (Variational Refinement)

把 3.4 的硬分割当初始化,跑变分 E-M 进一步收敛边界:

  • 模型:每个全局说话人 k 是一个潜变量高斯 N(μ_k, Σ_within),实体 e 的归属 z_e ∈ {1..K} 为潜变量;
  • E 步:算 r_{e,k} ∝ π_k · N(p_e | μ_k, Σ_within)(实体属各说话人的后验责任);
  • M 步:μ_k = Σ_e r_{e,k} p_e / Σ_e r_{e,k},更新 π_k;可选更新 Σ_within;
  • 迭代 em_iterations 轮;低置信实体用 r 软分配而非硬挂,边界更平滑。
这一步用生成式说话人模型"精修"3.4 的分割:把分割从一个组合优化解,变成在概率模型下的最大似然/变分推断,能纠正 3.4 个别误判。

3.6 歧义最优传输仲裁 (Ambiguity Optimal-Transport Arbitration)

对仍歧义(r_{e,·} 多峰、max 责任低)的实体,在 3.5 收敛的说话人集上求最优传输:

  • 代价 C(e,k) = 1 − r_{e,k},实体×说话人二分图;
  • Hungarian / Sinkhorn 求总代价最小、且满足"每实体唯一归宿、说话人容量下限"的分配;
  • 实在无解(全部低责任)→ 该实体自成新说话人,回到 3.5 再迭代一轮。

4. 整体流程

长音频
  │
 ├─ Step1 分段转写:切窗 → MOSS → 每窗 {offset, transcript(local [Sxx])}
 │
 ├─ Step2 滪窗提取:按 transcript 定位每实体语音区 → 滑窗 speaker encoder
 │     ↓ 每实体一组 sub-embeddings
 │
 └─ Step3 对应关系求解(算法集中):
       3.1 稳健原型(Tukey M-估计 → p_e;稳核剔离群;ρ_e/c_e;估 Σ_within)
       3.2 不足重提取("完全包含"独白段重 embed;还不足→待分配)
       3.3 生成式匹配概率(LLR + sigmoid → 同人后验 q_ef;估 Σ_between)
       3.4 最小违例分割(min Σ 拆开·q + 合并·(1−q);pivot 或 ILP 求解)
       3.5 变分 E-M(潜变量高斯,E/M 步迭代精修边界)
       3.6 最优传输仲裁(Hungarian/Sinkhorn 解歧义实体)
            ↓ (窗,local)→全局ID
       → 重写 merged.txt:全局时间戳 + 全局说话人

5. 输出

  • merged.txt:全局时间戳 + 全局说话人标签(跨窗一致)
  • prototypes.json:每实体原型 p_e + 置信度 c_e + 稳健度 ρ_e
  • posterior_graph.json:同人后验矩阵 q_ef(调试 3.3/3.4)
  • partition.json:3.4 分割 + 3.5 责任 r_{e,k}(调试收敛)
  • speaker_mapping.json:(窗,local)→全局 ID + 置信度 + 归属

7. 边界与限制

  • Σ_within / Σ_between 的估计质量决定 3.3 后验准不准;数据少时退化为裸余弦,建议外接预训练协方差或在校准集上拟合;
  • 3.4 的 pivot 是 3-近似,极端边界例可能次优;实体数少时用 ilp 取精确解;
  • 3.5 E-M 对初始化(3.4 的解)敏感;3.4 给的好初值是 E-M 收敛的前提;
  • 同一人跨窗音色漂移大 → 3.3 后验偏低 → 可能被 3.4 拆成两人;3.5 的软分配能部分缓解,极端需放宽 Σ_within;
  • MOSS 转写错把两人合一 → 该实体原型被污染;3.1 稳核能挡一部分,残留由 3.3 低后验自然隔离。
