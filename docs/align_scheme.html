MOSS 长音频说话人匹配方案(三步法)

把长音频(超 90min 上下文)的说话人一致化拆成三步:step1/step2 只做信息汇总,所有算法集中在 step3 的匹配逻辑。本版暂不使用前后两段重叠区的 1:1 对齐,改走"稳健原型 → 分层匹配"。

总览:
1分段转写:长音频切段,每窗跑 MOSS,得到带 local [Sxx] 的转写。
2滑窗提取 embedding:按转写定位每说话人语音区,滑窗过 speaker encoder,得大量 sub-embedding。
3说话人匹配(核心):稳健原型 → 不足重提取 → 亲和图 + 分层匹配(种子锚定 → 互最近邻 → 残留分配 → 冲突最优求解) → 全局说话人 ID。

1. 参数

参数默认含义
sub_window_seconds3.0step2 滑窗长度
sub_step_seconds1.0滑窗步长
min_sub_speech_seconds1.0子窗有效语音 < 此则跳过
stability_threshold0.73.1 稳核过滤:子向量到原型余弦 ≥ 此才留
min_stable_count3稳核子向量数 < 此 → 判不足,进 3.2
min_stable_speech_seconds5.0稳核总语音 < 此 → 判不足
reextract_min_solo_seconds3.03.2 只用 ≥ 此的"完全包含"独白段重提取
tau_spawn0.453.3 种子区分门:候选种子与已有种子最大亲和 < 此 → 立为新说话人锚
tau_match0.603.3 互最近邻匹配门:互为最近且亲和 ≥ 此 → 建同说话人边
tau_assign0.503.3 残留分配门:实体到种子亲和 ≥ 此才可分配,否则自成新说话人

1 Step1 分段转写

长音频切成 ≤90min 的窗(VAD 切静音或固定长),每窗独立跑 MOSS 推理。

  • 每窗输出 [start][Sxx]text[end],时间戳为窗内 local,说话人标签 [Sxx] 每窗独立从 S01 重新编号(窗1的 S01 与窗2的 S01 不一定是同一人——这是 step3 要解决的对应问题);
  • 记录 {offset, transcript};offset 用于还原全局时间戳。

本步只产出信息,不做任何匹配。窗是否重叠不影响 step3(本版不用重叠 1:1)。

2 Step2 滑窗提取 embedding

对每窗,按转写段定位每个 local 说话人的语音区间,在区间上滑窗过 speaker encoder。

对窗 i 的每个 local 说话人 L:
  语音区 = {(seg_start, seg_end) | seg.speaker == L}
  for t in seg_starts, step=sub_step:
      w = [t, t+sub_window]
      if w 内 L 的有效语音 >= min_sub_speech_seconds:
          emb = speaker_encoder(audio[ w + offset ])   # 归一化
          存 (i, L, t, emb, dur)  # 一个 sub-embedding
为什么滑窗而非整段一个 embedding:一个说话人在一窗里产生多个 sub-embedding,才能在 3.1 做稳健化(剔离群、判紧凑)。整段一个向量无法判稳。

encoder 选型:SpeechBrain ECAPA-TDNN(纯 pip,起步)或 pyannote/embedding(WavLM,质量更高),均归一化。不能用 Whisper encoder。

3 Step3 说话人匹配(核心,所有算法在此)

输入:每实体 e = (窗 i, local L) 的一组 sub-embeddings。输出:e → 全局说话人 ID。把"哪些实体是同一人"建模为亲和图上的分层匹配。

3.1 稳健原型 (Robust Prototype)

对每实体 e,把 sub-embeddings 蒸成一个稳健原型 p_e + 置信度 c_e:

  1. 迭代重加权 medoid:以均值 m 为参照,按余弦亲和给子向量赋权(亲和高→权大,离群→权小),重算 m,迭代收敛 → 原型 p_e;
  2. 稳核过滤:剔除 cos(emb_k, p_e) < stability_threshold 的离群(混人/噪声子窗),剩稳核;
  3. 稳健度 ρ_e = 稳核内子向量到 p_e 的平均亲和(高=音色紧凑可信);
  4. 置信度 c_e = g(ρ_e, 稳核规模, 稳核总语音时长)。

3.2 不足 / 缺失的重提取(完全包含的说话人)

若 稳核数 < min_stable_count 或 稳核总语音 < min_stable_speech_seconds → 判为不足:

  1. 在该窗找 "完全包含"的独白段:该段只有 L、无重叠、段长 ≥ reextract_min_solo_seconds;
  2. 对干净独白段重切音频、重滑窗 embed(可用更长 sub_window 提升信号),再跑一次 3.1 稳核过滤并入稳核。
仍不足 → c_e 极低,标记为待分配实体(不当种子,但可在 3.3 被分配)。极端无数据 → 留为独立 ID(待人工)。

3.3 分层匹配 (Layered Matching)

每实体已有原型 p_e + 置信度 c_e。构建亲和图 G=(V,E),V=实体,边权 a(e,f)=cos(p_e,p_f)。按下面四层依次求解,得到每实体归属的全局说话人。

① 种子锚定 (Seed Anchoring) —— 确立候选全局说话人锚
按 c_e 降序遍历实体作为候选种子;候选种子 s 若与已选种子的最大亲和 max a(s, ·) < tau_spawn → 列为新全局说话人锚(它是高置信的"新人");否则丢弃(已被某个已有种子代表)。结果:锚集 A = {a_1..a_K}。
② 互最近邻匹配 (Mutual-Nearest-Neighbor Matching) —— 建高置信同说话人边
对非种子实体 e,找其亲和最大者 nn(e);若 a(e, nn(e)) ≥ tau_match 且 nn(nn(e)) == e(互为最近) → 在 e 与 nn(e) 间建同说话人边。这些边是"两人互认"的强证据,几乎无误配。
③ 残留分配 (Residual Assignment) —— 把剩余实体挂到锚
未被②匹配的实体 e:分配给亲和最大、且 a(e, a_k) ≥ tau_assign 的种子锚 a_k;若对所有锚 a < tau_assign → e 自成新说话人(并入锚集 A)。
④ 冲突最优求解 (Conflict Resolution) —— 仲裁多对一/一对多
对"多实体争同一锚 / 一实体多候选"的冲突子图:以 cost(e, a_k) = 1 − a(e, a_k) 为代价建二分图(实体 × 锚),Hungarian 求总代价最小的分配。允许 many-to-one(多实体同属一锚=同人跨窗),但每实体归宿唯一。低置信实体在代价里被弱化(不主导仲裁)。

四层结束:每实体归属一个锚(或自成锚)→ 锚 = 全局说话人(S01, S02, …)→ 得 (窗, local) → 全局 ID 映射。

4. 整体流程

长音频
  │
 ├─ Step1 分段转写:切窗 → MOSS → 每窗 {offset, transcript(local [Sxx])}
 │
 ├─ Step2 滑窗提取:按 transcript 定位每实体语音区 → 滑窗 speaker encoder
 │     ↓ 每实体一组 sub-embeddings
 │
 └─ Step3 说话人匹配(算法集中):
       ├─ 3.1 稳健原型:迭代重加权 medoid → p_e;剔离群稳核;ρ_e/c_e
       ├─ 3.2 不足重提取:稳核不够 → "完全包含"独白段重 embed;还不足→待分配
       └─ 3.3 分层匹配:
            ① 种子锚定(c_e 降序,与已有锚 max<τ_spawn → 新锚)
            ② 互最近邻匹配(互为最近且 a≥τ_match → 同说话人边)
            ③ 残留分配(挂到 max a≥τ_assign 的锚,否则自成新锚)
            ④ 冲突最优求解(Hungarian 最小代价二分分配)
             ↓ (窗,local)→全局ID
       → 重写 merged.txt:全局时间戳 + 全局说话人

5. 输出

  • merged.txt:全局时间戳 + 全局说话人标签(S01/S02… 跨窗一致)
  • prototypes.json:每实体原型 p_e + 置信度 c_e + 稳健度 ρ_e
  • affinity_graph.json:实体间亲和矩阵 + 匹配边(调试 ①②③④)
  • speaker_mapping.json:(窗,local)→全局 ID + 置信度 + 归属锚

6. 与旧方案(重叠 1:1)的差异

维度旧方案(scheme.html)本方案
对应依据重叠区同时间 1:1 对应亲和图 + 分层匹配(互最近邻→残留→冲突)
是否用重叠区必须重叠且两窗都转写不用(暂不考虑重叠 1:1)
稳健性靠重叠同时间证据靠迭代重加权 medoid + 稳核剔离群
不足处理短段降权"完全包含"独白段重提取
冲突取最高余弦Hungarian 二分最优分配
结构逻辑分散算法集中 step3,step1/2 只汇总

7. 边界与限制

  • 三个门 tau_spawn / tau_match / tau_assign 需按你的 encoder + 域校准(不同 encoder 亲和尺度不同);
  • 同一人跨窗音色漂移(疲劳/换麦)可能被互最近邻②漏掉 → 走③残留分配兜底,仍挂到最相近的锚;
  • MOSS 转写把两人合一/一人拆二 → 该实体原型被污染;3.1 稳核过滤能挡一部分,极端的会成低置信待分配;
  • 本版不用重叠 1:1,实现简单;若后续互最近邻②命中率不够,可再叠加重叠同时间约束作 ②' 的强证据边。
