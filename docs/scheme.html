MOSS-Transcribe-Diarize 长音频推理方案

针对超过模型上下文(~90min)的长音频,采用 VAD 静音切窗 + 重叠 + 中点去重 + 跨窗说话人匹配 流水线,产出全局时间戳、全局一致说话人标签的完整转写。本方案含三项关键增强:①重叠区按说话人 1:1 对应合并、②剩余音频够短则整段推理、③短段 embedding 降置信/丢弃。

1. 参数

参数默认含义
target_minutes (T)20基准窗长(标称切点 0/20/40/60…)
tolerance_minutes (L)2切点搜索容差;重叠 ~2L = 4min
max_remaining_minutes30② 剩余 ≤ 此值则整段推理,不再切
min_gap_seconds1.0VAD 静音最短,过滤句中微停顿
min_segment_seconds10窗短于此则并入上窗或丢
short_embedding_seconds2.0③ 段短于此 → embedding 置信度降权
drop_embedding_seconds0.5③ 段短于此 → 直接丢弃,不参与 embedding
speaker_match_threshold0.6余弦判同人阈值(需校准)

2. 切窗算法(绝对标称锚定,不累积漂移)

标称切点在 0, 20, 40, 60…。每个边界在标称点 ±L 内用 VAD 找静音,头取最早(最靠标称-L)、尾取最晚(最靠标称+L),使两窗边界都落在 turn 间隙 → turn 完整。无 VAD 则强切。

窗1: start = 0,                                end = VAD∈[20,22] 取最靠近 22(最晚)
窗2: start = VAD∈[18,20] 取最靠近 18(最早),  end = VAD∈[40,42] 取最靠近 42(最晚)
窗3: start = VAD∈[38,40] 取最靠近 38(最早),  end = VAD∈[60,62] 取最靠近 62(最晚)
...
无 VAD → 头部强切 18(标称-2)、尾部强切 22(标称+2)

举例:
  窗1 end  = 21.5,  窗2 start = 18.5  →  重叠 [18.5, 21.5] ≈ 3min
  极端(头18/尾22)                   →  重叠满 4min
头取最早、尾取最晚 = 安全拉满:每窗尽量长、重叠尽量大,给"turn 跨切点"留最大缓冲(没人连讲 4min 不停顿,所以碎片化几乎不可能)。
② max_remaining: 规划下一窗前,若 剩余时长 D - cursor ≤ 30min,直接把剩余整段作为最后一窗(end=D,停止切)。避免末尾出现"20-25min,切了不如不切"的小窗。

3. 每窗推理

  • ffmpeg -ss start -t (end-start) -ac 1 -ar 16000 切出 16k mono wav
  • generate_transcription(do_sample=False, max_new_tokens=65536) 贪心解码(离线要质量)
  • 收集 {offset:start, end:end, transcript}(transcript 用 local 时间戳)

4. 文本 + 时间戳合并(merge_windows)

4.1 中点带去重

相邻窗重叠 ~3-4min。重叠区取中点 m,左半归左窗、右半归右窗;每段按起点归到唯一窗,整段保留(不切文本)。

窗1 end=E1, 窗2 start=S2, 重叠 [S2, E1], 中点 m=(S2+E1)/2
窗1 可信带 [.., m], 窗2 可信带 [m, ..]
段起点 < m  → 窗1 保留(整段),窗2 丢
段起点 ≥ m  → 窗2 保留(整段),窗1 丢
→ 不重、不漏、不切文本

4.2 直观例子:一段会议跨过重叠区

设窗1 end=21.5,窗2 start=18.5,重叠 [18.5, 21.5],中点 m=20.0。三段话(全局时间):

段(全局时间)起点在中点哪侧窗1 有?窗2 有?谁保留
A [18:50][S01]那我们看下一个议题[19:05]18:50左(<20:00)✅完整✅完整窗1
B [19:10][S02]性能优化我压到50ms[20:55]19:10左(<20:00)✅完整⚠️残段(19:10-20:55 头在窗1)窗1 完整版;窗2 残段丢
C [21:00][S03]测试报告明天给[21:30]21:00右(>20:00)❌(超窗1终 21:30)✅完整窗2
关键:窗1 切在 21:5(静音处)→ 段 B 在窗1 里是完整的(19:10-20:55 < 21:5)。窗2 也有 B 但是残段(从 18:5 才开始听,缺 18:5 之前的头)→ 被丢,留窗1 完整版。VAD 切静音 = turn 不截断,这是整个方案能干净合并的地基。

4.3 截断检测(兜底强切)

若某段 end 落在某窗 end 的 ε(0.3s)内 → 判为疑似强切截断 → 改留下一窗的完整版。这是无 VAD 命中时的兜底。

5. 跨窗说话人匹配(核心 ①)

目标:把每窗本地 [S01] 映射到全局一致 ID。利用重叠区做"同时间同说话人"的对齐证据。

5.1 每窗提取 speaker embedding

  • 按 MOSS 段的 [start][Sxx]text[end] 从音频切出每个说话人的语音段;
  • 每段过 speaker encoder → embedding;
  • 按 (窗, 本地标签) 聚合 → 加权均值(权重 = 置信度,见 §6)。

5.2 重叠区 1:1 对应(优先路径)

在相邻窗 N-1 / N 的重叠区,两窗都有段和说话人。把重叠区按说话人切开:
  1. 对重叠区里两窗各自的 local 说话人,各算 embedding + 置信度;
  2. 尝试 1:1 对应(Hungarian):若每对余弦 ≥ 阈值 → 认为这些 local 说话人就是同一全局说话人(重叠区=同时间,直接对齐,高置信);
  3. 若对不上(某对余弦 < 阈值,或人数不等)→ 说明某窗转写/切分有错 → 进入 5.3。

5.3 全局余弦重分配(回退路径)

对没对上的 local 说话人,跨全窗算 embedding 余弦,诊断哪部分错了:

  • 余弦 ≥ 阈值 → 合并到已有全局 ID;
  • 都低于阈值 → 开新全局 ID;
  • 冲突(一个 local 对上多个全局 / 多个 local 对上一个全局)→ 取最高余弦,其余开新 ID 或标记待人工复核。
逻辑:重叠区能对上的直接确认(强证据),对不上的退到全局相似度找错并重分配。这样既利用了重叠的同时间证据,又能诊断哪窗切错了。

6. 置信度与短段(③)

短段 embedding 不可靠,需差异化处理:
  • 段 < 2s → 降低置信度(聚合时降权,不主导该说话人向量);
  • 段 < 0.5s → 直接丢弃,不参与 embedding 聚合;
  • 置信度 = f(总语音时长, 段数, 平均段长, 自一致性);
  • 用途:重叠 1:1 是否采信(低置信 → 不采信,走 5.3);聚合权重;冲突时优先高置信。

7. 整体流程

长音频
  │
  ├─ silero-vad → 静音 gaps
  ├─ 切窗(标称锚定 + VAD 头最早/尾最晚 + max_remaining 整段兜底)
  │     ↓ 每窗 [start, end]
  ├─ ffmpeg 切 + generate_transcription → 每窗 transcript(local)
  │     ↓ windows = [{offset, end, transcript}]
  ├─ merge_windows.merge → 文本+时间戳合并(中点去重 + 截断检测)
  │     ↓ 全局时间戳文本(说话人仍是 local 标签)
  ├─ speaker encoder → 每段 embedding(+ 置信度,短段降权/丢)
  ├─ 重叠区 1:1 对应 → 高置信直接合并;对不上 → 全局余弦重分配
  │     ↓ 全局说话人 ID
  └─ 输出 merged.txt(全局时间戳 + 全局说话人)

8. 输出

  • merged.txt:全局时间戳 + 全局说话人标签的完整转写
  • windows.json:每窗 offset/end/transcript(调试)
  • speaker_mapping.json:(窗,本地)→ 全局 ID + 置信度

9. 边界与限制

  • silero 不可用 → 全程强切(标称±L),截断检测兜底;
  • 跨窗同说话人音色漂移(疲劳/换麦)可能误并,置信度低的不强合并;
  • 重叠区 1:1 依赖两窗转写都正确;一窗切错会触发 5.3 回退;
  • speaker encoder 须是 speaker-trained(如 ECAPA / pyannote),不能用 Whisper encoder(它编码内容不编码音色)。
